# Firmware migration: 1.0.36 to 1.0.37

## Release status

The three-run hardware gate in `docs/firmware-ota-contract.md` passed on
2026-07-16 for the final `1.0.37` bytes. The exact results are recorded in
`docs/control-center-hardware-test-evidence.md`. Merge, tag, and release remain
separate guarded actions.

## Why this migration needs a compatibility mode

Firmware `1.0.36` can accept an OTA TCP connection but stop draining upload
bytes. A normal macOS sender then buffers roughly `130 KB` before ending with a
broken pipe. The target `1.0.37` binary is not validated in this failure mode
because it never arrives completely.

The `1.0.36` RAW handler also leaves the ESP8266 `Update` object active after a
timeout. A second upload in the same boot can then fail with the misleading
message `Update failed: No Error`. A power restart clears that state.

## Customer-safe migration behavior

- Control Center downloads and validates the release artifact before upload.
- It pins the selected device and pauses the display stream.
- The Companion uses the paced RAW transport specified in
  `docs/firmware-ota-contract.md`.
- If the upload response is lost but `1.0.37` returns, the update is treated as
  successful.
- If `1.0.36` returns after an interrupted upload, Control Center stops. It must
  not attempt multipart or another upload in the same boot.
- Recovery does not require a data USB connection: disconnect VibeTV power for
  10 seconds, reconnect it, wait for the picture, and retry once.

## Changes included in the 1.0.37 receiver

- non-blocking partial socket reads instead of `readBytes()` for full 1 KB
  blocks;
- exact RAW body-size validation;
- unrelated TCP and UDP connections closed before flashing;
- renderer and filesystem resources released before update mode;
- one cleanup path for every ESP8266 updater failure;
- controlled restart after an OTA failure that entered update mode.

These changes protect `1.0.37 -> later` updates. They cannot repair devices
while they are still executing `1.0.36`; the paced Control Center sender is the
bridge for that installed base.
